# Команды для работы с Git и GitHub

## Первоначальная настройка (если репозиторий еще не создан)

### 1. Инициализация репозитория
```bash
git init
```

### 2. Добавление удаленного репозитория
```bash
git remote add origin https://github.com/ваш_username/WBagentforDasha.git
```

Или если используете SSH:
```bash
git remote add origin git@github.com:ваш_username/WBagentforDasha.git
```

### 3. Проверка удаленного репозитория
```bash
git remote -v
```

## Основные команды для работы

### Добавление файлов в индекс
```bash
# Добавить все измененные файлы
git add .

# Добавить конкретный файл
git add имя_файла.py

# Добавить все файлы определенного типа
git add *.py
```

### Создание коммита
```bash
git commit -m "Описание изменений"
```

Примеры:
```bash
git commit -m "Добавлен парсер продаж Wildberries"
git commit -m "Добавлена автоматизация скачивания отчётов в Excel"
git commit -m "Исправлена ошибка чтения .env файла"
```

### Отправка на GitHub
```bash
# Первая отправка (создание основной ветки)
git push -u origin main

# Последующие отправки
git push
```

Если ваша основная ветка называется `master`:
```bash
git push -u origin master
```

## Полный цикл работы

### 1. Проверить статус
```bash
git status
```

### 2. Добавить изменения
```bash
git add .
```

### 3. Создать коммит
```bash
git commit -m "Ваше описание изменений"
```

### 4. Отправить на GitHub
```bash
git push
```

## Полезные команды

### Просмотр истории коммитов
```bash
git log
```

### Просмотр изменений
```bash
# Показать изменения в рабочей директории
git diff

# Показать изменения в индексе
git diff --staged
```

### Отмена изменений
```bash
# Отменить изменения в файле (до добавления в индекс)
git checkout -- имя_файла

# Убрать файл из индекса (но сохранить изменения)
git reset HEAD имя_файла

# Отменить последний коммит (сохранив изменения)
git reset --soft HEAD~1
```

### Просмотр веток
```bash
# Показать все ветки
git branch

# Создать новую ветку
git branch имя_ветки

# Переключиться на ветку
git checkout имя_ветки
```

### Получение изменений с GitHub
```bash
# Скачать изменения (без слияния)
git fetch

# Скачать и объединить изменения
git pull
```

## Быстрый старт для первого пуша

```bash
# 1. Инициализация (если еще не сделано)
git init

# 2. Добавление удаленного репозитория (если еще не добавлен)
git remote add origin https://github.com/ваш_username/WBagentforDasha.git

# 3. Добавить все файлы
git add .

# 4. Создать первый коммит
git commit -m "Initial commit: парсер продаж Wildberries"

# 5. Отправить на GitHub
git push -u origin main
```

## Настройка Git (если еще не настроено)

```bash
# Установить имя пользователя
git config --global user.name "Ваше Имя"

# Установить email
git config --global user.email "ваш_email@example.com"

# Проверить настройки
git config --list
```

## Игнорирование файлов

Файл `.gitignore` уже создан и включает:
- `.env` (секретные данные)
- `*.json` (данные)
- `*.xlsx` (отчёты)
- `__pycache__/` и другие служебные файлы Python

## Работа с секретами

**ВАЖНО:** Файл `.env` с токеном НЕ должен попадать в репозиторий!

Проверка перед коммитом:
```bash
# Проверить, что .env не добавлен
git status

# Если .env в списке, убрать его
git reset HEAD .env
```

## Решение проблем

### Если забыли добавить файл в последний коммит
```bash
git add забытый_файл.py
git commit --amend --no-edit
```

### Если нужно изменить последний коммит
```bash
git commit --amend -m "Новое описание"
```

### Если нужно отправить изменения в другую ветку
```bash
git push origin имя_ветки
```

### Если возник конфликт при pull
```bash
# Посмотреть конфликты
git status

# После решения конфликтов
git add .
git commit -m "Решение конфликтов"
git push
```
